    # add bounding box to color image
    cv2.rectangle(img,(x,y),(x+w,y+h),(255,0,0),2)

# display the image, along with bounding box

plt.show()

Number of faces detected: 1

Before using any of the face detectors, it is standard procedure to convert the images to grayscale. The detectMultiScale function executes the classifier stored in face_cascade and takes the grayscale image as a parameter.

In the above code, faces is a numpy array of detected faces, where each row corresponds to a detected face. Each detected face is a 1D array with four entries that specifies the bounding box of the detected face. The first two entries in the array (extracted in the above code as x and y) specify the horizontal and vertical positions of the top left corner of the bounding box. The last two entries in the array (extracted here as w and h) specify the width and height of the box.

Write a Human Face Detector¶

We can use this procedure to write a function that returns True if a human face is detected in an image and False otherwise. This function, aptly named face_detector, takes a string-valued file path to an image as input and appears in the code block below.

# returns "True" if face is detected in image stored at img_path
def face_detector(img_path):
    img = cv2.imread(img_path)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    faces = face_cascade.detectMultiScale(gray)
    return len(faces) > 0

Question 4: Outline the steps you took to get to your final CNN architecture and your reasoning at each step.

Answer: Arriving on a final architecture involved a lot of trial and error beyond a general intuition at the beginning. Following are some of the steps I took in progression to arriving at a final model which provided a mediocre 11% testing accuracy.

  • Initially I started of with a network comprising of 4 convolution layers, with a 3x3 filter size and geometrically increasing number of filters, much like the final architecture. However, having upto 256 filter maps in the final layer, slowed the training of the network and I removed it. Going ahead, it could be added to improve accuracy.
  • In the initial architecture, I had 3 fully-connected layers and this lead to a number of unprecedented problems. My intial reasoning for having 3 FC layers was that it might not be a great idea to reduce number of features from 100352 to 500 in the same layer as there may be a loss in information. however, having another FC intermediate layer increased the number of weights being calculated exponentially and the space required to save the model went up from 200 mb to over 1.5 GB which is impossible to store with availbe space on the GPU enabled workspace. Removing this extra layer helped to kickstart my model and start training.
  • Next, I tried to experiment with the stride of the kernels and the padding required accordingly. By comparing with the required dimensions in each layer, I arrived at the final values.
  • At this stage, I was still obtaining a test accuracy of only 8%. I realized that using 2 dropout layers like I did initially might not be necessary as I did not have such a deep network and there was no chance of overfitting my data when the learning was already so slow. I omitted these layers and it improved my performance considerably.

Beyind this, I tried differeent size for each layer without much reasoning, just to understand how my results are affected by it. I also tuned some hyperparameters such as number of epochs and the learning rate. Of course I could improve my model a lot more, but I stopped trying beyond reaching a minimum required accuracy in lieu of the time available for me at the moment. I would love to work harder on this architecture and improve it further.

def train(n_epochs, loaders, model, optimizer, criterion, use_cuda, save_path):
    """returns trained model"""

    valid_loss_min = np.Inf 
    
    for epoch in range(1, n_epochs+1):
        # initialize variables to monitor training and validation loss
        train_loss = 0.0
        valid_loss = 0.0
        
        ###################
        # train the model #
        ###################
        model.train()
        for batch_idx, (data, target) in enumerate(loaders['train']):
            # move to GPU
            if use_cuda:
                data, target = data.cuda(), target.cuda()
            ## find the loss and update the model parameters accordingly
            ## record the average training loss, using something like
            ## train_loss = train_loss + ((1 / (batch_idx + 1)) * (loss.data - train_loss))
            optimizer.zero_grad()
            # forward pass: compute predicted outputs by passing inputs to the model
            output = model(data)
            # calculate the batch loss
            loss = criterion(output, target)
            # backward pass: compute gradient of the loss with respect to model parameters
            loss.backward()
            # perform a single optimization step (parameter update)
            optimizer.step()
            # update training loss
#             train_loss += loss.item()*data.size(0)
            train_loss = train_loss + ((1 / (batch_idx + 1)) * (loss.data - train_loss))

        ######################    
        # validate the model #
        ######################
        model.eval()
        for batch_idx, (data, target) in enumerate(loaders['valid']):
            # move to GPU
            if use_cuda:
                data, target = data.cuda(), target.cuda()
            ## update the average validation loss
            
            output = model(data)
            # calculate the batch loss
            loss = criterion(output, target)
            # update average validation loss 
#             valid_loss += loss.item()*data.size(0)
            valid_loss =valid_loss + ((1 / (batch_idx + 1)) * (loss.data - valid_loss))
    
#         train_loss = train_loss/len(train_loader.sampler)
#         valid_loss = valid_loss/len(valid_loader.sampler)
        
        # print training/validation statistics 
        print('Epoch: {} \tTraining Loss: {:.6f} \tValidation Loss: {:.6f}'.format(
            epoch, 
            train_loss,
            valid_loss
            ))
        
        ## TODO: save the model if validation loss has decreased
        
        if valid_loss <= valid_loss_min:
            torch.save(model.state_dict(), save_path)
            valid_loss_min = valid_loss
            
    # return trained model
    return model

def test(loaders, model, criterion, use_cuda):

    # monitor test loss and accuracy
    test_loss = 0.
    correct = 0.
    total = 0.

    model.eval()
    for batch_idx, (data, target) in enumerate(loaders['test']):
        # move to GPU
        if use_cuda:
            data, target = data.cuda(), target.cuda()
        # forward pass: compute predicted outputs by passing inputs to the model
        output = model(data)
        # calculate the loss
        loss = criterion(output, target)
        # update average test loss 
        test_loss = test_loss + ((1 / (batch_idx + 1)) * (loss.data - test_loss))
        # convert output probabilities to predicted class
        pred = output.data.max(1, keepdim=True)[1]
        # compare predictions to true label
        correct += np.sum(np.squeeze(pred.eq(target.data.view_as(pred))).cpu().numpy())
        total += data.size(0)

Answer: To tinker with a pre-trained model, I had to first study the model, the data it was trained on and the similarities with the current task at hand. As we had learnt from the lessons in this course, the first step is to compare the size and similarity of the dataset on which the model was trained with the data that is going to be trained. I chose to use a pre-trained VGG16 model trained on ImageNet. From this I learned that the data of dog files we had are mostly a subset of the ImageNet classes, but much less in number compared to the size of ImageNet training data. Since we have small, but similar dataset to ImageNet I decided that it would be best to freeze all feature extraction layers during the training and only alter the fully-connected layers. The fully-connected layers only had to be changed to accomodate the new set of classes it had to predict which changed from 1000 to the 133 dog breeds we have. This was ennough to achieve an accuracy of 86% on our test data.
